# @упоминания: сопоставление людей Redmine ↔ MAX

Скрипт: `scripts/user_mapping_cli.py`

Чтобы пингнуть человека в чате, нужен его **user_id в MAX**. В событии
из Redmine лежит только `assigned_to.id` — это **другой** идентификатор,
из другой системы. Связи между ними не существует, поэтому людей
сопоставляем руками. Один раз, их немного.

## Где взять MAX user_id

**Искать руками не надо, спроси у CLI:**

```bash
uv run python scripts/user_mapping_cli.py max-members
```

```
MAX_USER_ID    ИМЯ                     USERNAME             ЧАТ
252123521      Leonid                                       D-Telecom NOTIFY
359199650      D-Telecom NOTIFY [BOT]  id2311313017_1_bot   D-Telecom NOTIFY
```

Нужен первый столбец.

> **Главное ограничение.** MAX не отдаёт «всех пользователей». Бот видит
> **только участников чатов, где он сам состоит**. Значит: **чтобы
> упомянуть человека, он должен быть в чате**. Нет в списке — добавь
> его в чат и повтори команду.

Это не наше ограничение, а логика мессенджера: упомянуть того, кого
в чате нет, невозможно.

## Где взять Redmine user_id

```bash
uv run python scripts/user_mapping_cli.py redmine-users
```

```
REDMINE_USER_ID    ИМЯ
5                  Леонид Клунный
10                 Максим Мерзляков
```

Список собирается **из задач** (авторы и исполнители), а не из
`/users.json`: тот требует прав администратора, которых у сервисного
юзера намеренно нет.

## Сопоставить

```bash
uv run python scripts/user_mapping_cli.py add \
    --redmine-user-id 10 --max-user-id 252123521
```

Подпись для упоминания подтянется из MAX сама. Нужна другая — задай явно:

```bash
uv run python scripts/user_mapping_cli.py add \
    --redmine-user-id 10 --max-user-id 252123521 --max-name "Макс"
```

## Посмотреть и проверить

```bash
uv run python scripts/user_mapping_cli.py list

uv run python scripts/user_mapping_cli.py list-redmine --redmine-user-id 10
```

`list-redmine` отвечает на вопрос «кого дёрнут, если задача упадёт
на этого человека».

## Удалить

```bash
uv run python scripts/user_mapping_cli.py remove \
    --redmine-user-id 10 --max-user-id 252123521
```

## Многие-ко-многим — зачем

Связь не 1:1, и это специально:

```bash
# Задача на Петю дёргает и Петю, и тимлида
add --redmine-user-id 10 --max-user-id 111
add --redmine-user-id 10 --max-user-id 222

# Петя в отпуске: его задачи пингуют Васю
add --redmine-user-id 10 --max-user-id 222   # Петя -> Вася
add --redmine-user-id 11 --max-user-id 222   # Вася -> Вася
```

Убрать «отпускное» правило = один `remove`, трогать код не нужно.

## Где упоминания появляются

- 🆕 новая задача
- ⏰ напоминание о дедлайне

В сменах статуса и комментариях — нет (решено 15.07.2026, чтобы не
частить пингами).

Упоминается **исполнитель**, не автор: автор и так знает, что создал
задачу.

## Что если человек не сопоставлен

Уведомление придёт **без пинга**, молча. Это нормальный режим:
подрядчик, уволившийся, робот-учётка. Ошибок в логах не будет.

## Формат упоминания (справочно)

Для живых людей работает **только** markdown-ссылка:

```
[Имя](max://user/252123521)
```

`@username` подсвечивается лишь у **ботов** — у людей поля `username`
в MAX просто нет. Проверено экспериментом на живом API.
