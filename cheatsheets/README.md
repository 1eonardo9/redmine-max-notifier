# Шпаргалки

Короткие памятки по эксплуатации сервиса. Не документация — именно
шпаргалки: команда, что делает, чем чревато.

| Файл | Про что |
|---|---|
| [routing.md](routing.md) | Куда слать уведомления: проект → чат MAX |
| [mentions.md](mentions.md) | @упоминания: где взять ID и как сопоставить людей |
| [templates.md](templates.md) | Правка шаблонов сообщений и проверка в чате |
| [operations.md](operations.md) | Запуск, миграции, логи, диагностика |

Всё запускается из корня репозитория. Настройки берутся из `.env`
(см. `.env.example`).

## Что нужно сделать один раз

```bash
uv sync                                     # зависимости
cp .env.example .env                        # и подставить свои значения
uv run python scripts/build_ca_bundle.py    # CA-bundle для MAX
uv run alembic upgrade head                 # схема БД
```

Дальше — [routing.md](routing.md): без маршрута уведомления никуда
не поедут.

## Первый запуск ничего не пришлёт

Это не поломка. Пустая таблица `polling_state` означает холодный старт:
первый цикл запоминает, где сейчас Redmine, и молчит. Иначе сервис
вывалил бы в чат всё, что попало в окно опроса. Уведомления пойдут
со второго цикла — примерно через минуту.

В логе видно так:

```
холодный старт: baseline issue_id=1234 journal_id=5678, уведомления не отправляются
```
